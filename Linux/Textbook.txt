ターミナルの操作方法
-> コマンドラインの操作方法
    ・左に一文字移動する：　Ctrl + b / <- + b
    ・右に一文字移動する：　Ctrl + f / -> + f
    ・行末に移動する：　Ctrl + a
    ・先頭に移動する：　Ctrl + e

-> 文字の削除
    文字を削除する；バックスペースキー
    カーソルで選択しているところを削除する：デリートキー
    単語単位で削除する：Ctrl + w

-> カットとヤンク(切り取りと貼り付け)
    カーソル位置から行末までの切り取り：　Ctrl + k
    カーソル位置から行頭までの切り取り：Ctrl + u
    ヤンク：　Ctrl + y

-> Tabの使い方
    足りない単語やパスを補完することができる


絶対パスと相対パス
->絶対パス
    ルートディレクトリからのパス
    メリットとして、曖昧さをなくして表示することができる
    デメリットとして、パスが長くなってしまい、マシンのディレクトリ構成に依存してしまう

    
    ->相対パス
    カレントディレクトリからのパス
    メリットとして、パスが短くなり、マシンのディレクトリ構成に依存しない
    ../  :親ディレクトリを示す
    ./   :カレントディレクトリを示す


ハードリンクとシンボリックリンク
->リンク
    ファイルに別名をつけることができる機能

->ハードリンク
    一つのファイルに複数の名前をつけることができる
    ディレクトリに対して設定できなかったりと不便なためあまり利用されない

->シンボリックリンク
    リンク先のパス名が書かれた小さな特殊ファイル
    ファイルを参照するためのファイルを新しく作成する。

パーミッション(アクセス権限)
->オーナー
    ファイルの所有者
    ファイルへのアクセス権限を自由に設定することができる
->グループ
    ユーザをまとめた集まりのこと
    グループ単位でアクセス権限を設定することができる

コマンドの利用方法
->オプション
    コマンドラインにハイフンで始まる引数を追加することでデフォルトとは違う動作をするようにするための機能
    オプションを検索するためのコマンドとして--helpオプションをつけるとオプションについて調べることができる


->pwd
    print working directory
    カレントディレクトリのパスを表示する

->cd [移動先のパス]
    change directory
    ディレクトリの変更をする
    オプションをつけないとホームディレクトリに移動する

->ls
    list
    指定したディレクトリの内容をリストアップする
    -> パス名展開
        ワイルドカードなどを利用して特定のパス名を選択することができる
        ex) ls ssh*
        
->mkdir [ディレクトリ名]
    make directory
    新規ディレクトリを作成する
    ->オプション
        -p ：階層型のディレクトリを作成するオプション(parent)

->touch
    新規ファイルの作成する

->rm
    ファイルの削除をする
    ->オプション
        -r: 再帰的にディレクトリツリーを削除することができる

->rmdir
    ディレクトリの削除をする

->cat
    concatenate
    ファイルの中身を確認する
    複数のファイルを指定すると複数のファイルを連結して表示することができる

->less

    表示した内容をスクロールをすることで確認することができる
    一画面下にスクロールする：スペースキー
    一画面上にスクロールする：b
    一行単位で下にスクロールする：j
    一行単位で上にスクロールする：k
    スクロール表示を終了する：q

->cp <copyしたい名前> <copy後の名前>
    copy
    ファイルやディレクトリをコピーする
    copy語の名前の箇所にディレクトリ名を指定するとそこにコピーが作成される
    ->オプション
        -r: ディレクトリをコピーする

->mv
    move
    ファイル名を変更する。ファイルを別のディレクトリに変更することもできる

->ln
    link
    リンクを作成する
    デフォルトで作成されるのは、ハードリンクである
    ->オプション
        -s: シンボリックリンクを作成する

->man
    manual
    オプションの詳細を確認することができる

->history
    過去に使用したコマンドを閲覧することができる
    インデントを利用することで、コマンドをコピペする必要がなくなる
    !インデント番号: 過去の実行したコマンドを実行することができる

->find
    ファイルやディレクトリを見つけ出す
    カレントディレクトリから下層を検索する

->locate
    findコマンドよりも高速にファイルを検索することができる
    デフォルトで入っていない場合があり、追加インストールが必要になる可能性がある
    ファイルのデータベースを作成されており、findコマンドよりも高速にファイルを検索することができる

->grep <検索文字列> <検索するファイル>
    global regular expression print
    ファイルの中身から特定の文字列を検索することができる

->chmod <誰に><どうするのか><何を> <対象ファイル/ディレクトリ>
    change mode
    パーミッションの変更をすることができる
    ->(シンボルモード)
    誰に：u(オーナー)/ g(グループ)/ o(その他のユーザ)
    どうするのか：r(読み取り)/ w(書き込み)/ x(実行)
    何を：+(権限の追加)/ -(権限の削除)/ =(指定したw権限と等しくする)
    
    ->(数値モード)
    パーミッションを数値で表す
    r=4, w=2, x=1で表すことができる
    ex) 755 -> rwxr-xr-x: このように表すことができる

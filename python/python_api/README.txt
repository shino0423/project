APIの学習まとめ

・APIとは
　Application Programming Interfaceの略
　他社が提供するサービス内の情報や機能を扱えるようにする仕組みとなる

・RESTfull API
 RESTの原則から成り立っているAPIを指す。
  -> RESTの原則
    ・アドレス可能性:一意のURIですべての情報を取得することが可能
    ・ステートレス性:全てのリクエストが完全に分離し、セッションなどの状態管理は行われないこと
    ・接続性:情報に別の情報をアクセスできるようにすること
    ・統一インターフェース:HTTPメソッドを利用すること


・メリットとデメリット
    ・メリット
        - 全ての機能を用意する必要がないため、開発にかかる時間・コストを削減できる
        - 起業がAPIを提供している場合が多いので、ハイクオリティな機能を実装できる
        - セキュリティの向上を図ることができる
        - システム変更による影響の最小化
    ex) amazon Pay
    
    ・デメリット
    - API提供サービスに依存する

・APIエコノミー
　APIを利用することで経済が活発になる経済状況を指す
　ex) 家計簿アプリやオンライン決済サービス

・APIを叩く上で必要な情報
　・APIに接続するための情報
    ->API登録のタイミングでAPIキーyやAPIエンドポイントを入手し、接続できるようになる
　・APIに送信するための情報
    ->どのHTTPメソッドを利用するか、ヘッダーのデータ種類やAPIキーなどの認証情報
    　JSON形式で送受信したいデータを記載することでAPIにデータを送信することができる

・JSONとは
　JavaScriptObjectNotationの略。JavaScriptのオブジェクトの表記方法をもとにしたデータフォーンマットであり、XMLなどと同様のテキストベースのデータフォーマットである

・HTTPメソッド
　クライアント(API使用者/ユーザ)が行いたい処理をサーバ(API側)に伝える役割
　->よく利用するHTTPメソッド
    - GET:取得
    - POST：作成
    - PUT：更新
    - DELETE：削除
